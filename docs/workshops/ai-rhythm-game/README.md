# AI Rhythm Game Workshop

A 90-minute offline workshop for Australian junior high students using Airbotix
Playground to create an original pop-star monster-battle rhythm game.

## Workshop Promise

Students leave with a playable mini rhythm game prototype and a clear
understanding of how AI can help with game design when humans still set the goal,
test the result, and make responsible creative choices.

## Audience

- Years 7-9, mixed ability
- 6 students maximum
- Best run as 3 pairs, one device per pair
- Beginner-friendly; no prior coding required

## Product Fit

The workshop uses the Creative Code Studio flow implemented in this app:

- Landing prompt: students describe the game idea.
- Generating screen: the playground builds the starter.
- Workspace: students use Chat, Code Editor, Game Runner, and Asset Viewer.
- Split layout is recommended for teaching because students can see the guide,
  code, and runner more predictably.
- Game Runner is used for frequent playtesting. Students press Play after each
  meaningful change.
- Asset generation can create or remix original stage art, character art, sound
  cues, or visual feedback where the backend is available. In offline-demo mode,
  the deterministic stub still supports the teaching pattern.

## Positioning

For parents:

Turn pop-culture excitement into practical AI and game-making skills. The class
is supervised, small-group, and outcome-based: every student pair builds, tests,
and presents a game.

For schools:

A compact STEAM incursion linking AI literacy, Digital Technologies, Media Arts,
design thinking, computational thinking, user testing, and responsible creation.

For students:

Build your own beat-battle dance game. Design original heroes, stage effects,
monsters, combos, and power moves, then playtest it with friends.

## Learning Outcomes

By the end, students can:

- Create a simple playable rhythm game using prompts and iteration.
- Explain core rhythm-game mechanics: beat, input timing, score, combo, feedback,
  difficulty, and win condition.
- Use AI as a creative assistant without treating it as an autopilot.
- Improve a game through testing, debugging, and peer feedback.
- Keep an inspired work original by changing names, characters, costumes, songs,
  lyrics, logos, and scene details.

## Success Criteria

Every pair should finish a prototype with:

- A clear original game title.
- A rhythm input loop using keys, taps, or timed prompts.
- Scoring or combo feedback.
- Hit/miss visual feedback.
- A 30-60 second playable round or short challenge.
- At least one improvement made after testing.

## Safety And Copyright Guardrails

The theme can draw on the broad trend of pop-concert fantasy action, but the
workshop must not use Netflix branding, movie titles in public marketing,
character names, exact costumes, lyrics, soundtrack clips, screenshots, logos, or
lookalike assets.

Student-safe prompt rules:

- No personal information.
- No real student photos, voices, full names, school details, or contact details.
- No gore, horror, sexualised dance content, or frightening demon imagery.
- Use "shadow beats", "stage monsters", or "fantasy rivals" for school-safe
  conflict.
- If an AI output looks too close to a known film, character, celebrity, or logo,
  change it.

## Recommended Setup

- 3 laptops or tablets, ideally one per pair.
- Teacher machine connected to a projector.
- Airbotix app preloaded and signed in with test/student accounts.
- Creative Code Studio opened at `/learn/playground/new`.
- Printed prompt cards and design cards.
- Optional headphones if sound generation or music loops are used.
- Timer visible to the class.
- Backup starter prompt ready in case a pair freezes.

## Research Notes

- Trend assumption: the student hook is informed by the 2025 Netflix film *KPop
  Demon Hunters*, which Netflix describes as a PG animated film about K-pop
  performers protecting fans from supernatural threats. Public workshop copy
  should still avoid official titles, character names, songs, lyrics, logos, and
  lookalike visuals unless a licence exists.
- Curriculum fit: the Australian Curriculum's AI connection frames Digital
  Technologies as helping students understand AI as a digital system involving
  data, algorithms, and purposeful digital solutions.
- Safety fit: eSafety Australia frames generative AI as a technology that can
  create text, image, audio, video, and simulations, and highlights the need for
  child-safety safeguards. This supports the workshop's closed, supervised,
  no-personal-data approach.
